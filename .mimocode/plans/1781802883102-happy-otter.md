# Scheduler RPM pacing: edge case fixes

## Context
User had 65k strings, 6 threads × 3 keys, RPM=15 per key. All 18 workers fired simultaneously → 429s → key "died". We added `_initial_stagger`, per-thread `worker_last_request` tracking, and stagger_offset in `_pace_delay`. Tests pass, but the explore audit found edge cases.

## Bugs to fix

### 1. Sweep phase doesn't clear `worker_last_request` (scheduler.py:1044)
After key death, sweep workers reuse `worker_idx` values 0,1,2… which may collide with main-phase workers that have recent `worker_last_request` entries. This causes an unnecessary pacing delay at sweep start.

**Fix**: Add `self.worker_last_request.clear()` inside the `with self.cond:` block at line 1044, next to `self.in_flight = 0` and `self.in_flight_workers.clear()`.

### 2. `_pace_delay` overwrites `_translate_with_retries` timestamps (scheduler.py:887)
`_translate_with_retries` records `worker_last_request[worker_idx] = time.time()` at lines 609/628 (success/error paths). Then `_pace_delay` immediately overwrites it at line 887 with `time.time()`. The writes at 609/628 are **only useful for the 429 cooldown** (line 668), but they're always overwritten before the next cycle. The code is correct but confusing.

**Fix**: Remove the writes at lines 609 and 628. Instead, record `worker_last_request` only in two places:
- In `_pace_delay` at line 887 (for pacing between cycles)
- In the 429 cooldown path at line 668 — use `time.time()` directly instead of `worker_last_request.get(worker_idx, time.time())` since the request just happened

This simplifies the dual-purpose naming and makes the data flow clear.

### 3. Unpause resets target without stagger (scheduler.py:893)
After unpause, `target = time.time() + self.delay_seconds` — stagger_offset is lost. All threads re-synchronize.

**Fix**: After unpause, restore the stagger by recording the time and using the same logic as the main path:
```python
last = self.worker_last_request.get(worker_idx)
target = (last + self.delay_seconds) if last is not None else time.time() + self.delay_seconds
```
This way after unpause, pacing resumes from the last recorded request, preserving natural spacing.
